# hallucinate-embedding

A tiny Cloudflare Worker that pretends to be an embedding API by asking a language model to generate a deterministic-looking 128-dimensional vector for a given text input.

This project is experimental, weird, and intentionally fake. It does **not** use a real embedding model. Instead, it prompts an LLM to “behave like an embedding model” and return 128 comma-separated floating-point values between `-1` and `1`.

## What it does

`hallucinate-embedding` accepts text through either a `GET` query parameter or a raw `POST` body, then returns a JSON array of 128 numbers.

Example response:

```json
[0.12, -0.44, 0.03, 0.91, ...]
```

The Worker:

1. Fetches the current model name from:

```txt
https://best-model.api-cloud-flare.workers.dev/
```

2. Sends the input text to Cloudflare Workers AI using `env.AI.run`.

3. Prompts the model to output exactly 128 floats.

4. Parses the comma-separated output into numbers.

5. Pads missing values with `0` if fewer than 128 numbers are returned.

## Response

The API returns a JSON array with exactly 128 numeric values.

```json
[
  0.104,
  -0.218,
  0.441,
  0.002
]
```

Actual responses contain 128 elements.

## Why?

Because sometimes you want an embedding-shaped thing without using a real embedding model.

Possible uses:

* Experiments
* Demos
* Toy semantic-search projects
* Testing vector pipelines
* Chaos engineering for AI infrastructure
* Making people uncomfortable

## Important warning

These are **not real embeddings**.

The output is generated by a chat/completion model following a prompt. That means:

* Semantic quality is not guaranteed.
* Numeric stability is not guaranteed.
* Determinism is best-effort only.
* Different models may produce different vectors.
* The model may return malformed output.
* This should not be used for serious retrieval, ranking, clustering, or ML workloads.

For production use, use an actual embedding model.

## How it works

The core prompt instructs the model to return a vector:

```txt
you behave like and embedding model and return a vector representing the semantic value of provided text. Output exactly 128 floats, range -1 to 1, comma-separated, no words, no explanation. Represent semantic content of input as numbers deterministically.
```

If the first request fails, the Worker retries using a simplified version of the input text where duplicate whitespace-separated words are removed.

```js
[...new Set(text.split(/\s+/))].join(' ')
```

If the model returns fewer than 128 values, the Worker fills the remaining dimensions with zeroes.

```js
const arr = Array(128).fill(0).map((_, i) => vec[i] || 0);
```

## Cloudflare setup

This Worker expects a Cloudflare Workers AI binding named `AI`.

Example `wrangler.toml`:

```toml
name = "hallucinate-embedding"
main = "src/index.js"
compatibility_date = "2024-01-01"

[ai]
binding = "AI"
```

Deploy with:

```bash
npx wrangler deploy
```

## Example Worker

```js
import { env } from "cloudflare:workers";

const getModel = async () => {
  try {
    const res = await fetch("https://best-model.api-cloud-flare.workers.dev/");
    const data = await res.json();
    return data.model;
  } catch (e) {
    return String(e);
  }
};

const getVec = async (model, text) => {
  try {
    const resp = await env.AI.run(model, {
      messages: [
        {
          role: "system",
          content:
            "you behave like and embedding model and return a vector representing the semantic value of provided text. Output exactly 128 floats, range -1 to 1, comma-separated, no words, no explanation. Represent semantic content of input as numbers deterministically."
        },
        { role: "user", content: text }
      ],
      temperature: 0,
      seed: 42
    });

    return resp.response.trim();
  } catch (e) {
    try {
      const resp = await env.AI.run(model, {
        messages: [
          {
            role: "system",
            content:
              "you behave like and embedding model and return a vector representing the semantic value of provided text. Output exactly 128 floats, range -1 to 1, comma-separated, no words, no explanation. Represent semantic content of input as numbers deterministically."
          },
          {
            role: "user",
            content: [...new Set(text.split(/\s+/))].join(" ")
          }
        ],
        temperature: 0,
        seed: 42
      });

      return resp.response.trim();
    } catch (e) {
      return String(e);
    }
  }
};

let model;

async function fakeEmbed(text) {
  if (!model) {
    model = getModel();
  }

  model = await model;

  const raw = await getVec(model, text);
  const vec = raw.split(",").map(Number);
  const arr = Array(128)
    .fill(0)
    .map((_, i) => vec[i] || 0);

  return arr;
}

export default {
  async fetch(request) {
    let text;

    try {
      if (request.method === "GET") {
        text = new URL(request.url).searchParams.get("text");
      } else {
        text = await request.text();
      }
    } catch (e) {
      text = String(e);
    }

    const result = await fakeEmbed(text);

    return new Response(JSON.stringify(result), {
      headers: {
        "Content-Type": "application/json"
      }
    });
  }
};
```

## Limitations

This project is intentionally fragile.

Known limitations:

* The model may ignore formatting instructions.
* `NaN` values can appear if the model emits non-numeric text.
* Empty input may still produce a vector.
* The output may not be meaningfully comparable across requests.
* The selected model depends on an external endpoint.
* The returned vector is only as reliable as the model’s compliance with the prompt.
